# Repo Boundary

## Owns

`mesh-ecology-testbed` owns:

- a controlled local participation testbed
- explicit tutorial concerns, actors, and scenarios
- local observation records for publications and responses
- a lightweight browser-facing inspection surface
- probe/test documentation for what the local harness does and does not prove

## Does Not Own

`mesh-ecology-testbed` does not own:

- primitive runtime semantics
- discovery semantics
- replication semantics
- writer admission doctrine
- deployment/profile canon
- reusable pack taxonomy
- app-specific canon for any consuming mesh app

## Upstream Boundary

`mesh-ecology` remains authoritative for:

- mesh participation posture
- actor hygiene
- canonical vs auxiliary vs probe/test distinction
- the rule that cross-runtime truth must come through supported participation surfaces

`mesh-ecology-packs` remains authoritative for:

- packaging posture
- shared deployment vocabulary
- reusable candidate actor conventions
- control-plane and adjacent-repo language

## Boundary Consequence

This repo stays adjacent.

It can simulate:

- local publication acceptance
- tutorial actor responses
- delayed and partial observations
- plurality and anomaly patterns

It must not imply:

- that these observations are mesh truth
- that this harness is a runtime substitute
- that passing scenarios upgrades a consumer into canonical mesh posture
